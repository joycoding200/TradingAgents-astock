# Issue 011 - 市场分析师报告编造股票别名

- **状态**: 🟡 待修复（不影响决策，仅显示问题）
- **发现版本**: v0.2.27
- **发现日期**: 2026-07-20

## 问题描述

运行 300131（英唐智控）完整分析时，市场分析师（market_analyst）的报告标题中出现错误别名：

```
# 🟦 300131 英唐智控（世纪瑞尔）技术面分析报告
```

「世纪瑞尔」对应的是另一只 A 股（300150），而非 300131 英唐智控。这是 LLM 幻觉，系统代码中没有任何地方存在「世纪瑞尔」这个字符串。

## 根因

市场分析师的系统提示词未明确禁止 LLM 编造别名/曾用名。DeepSeek 模型在生成报告时，自行联想了一个错误的曾用名并放入标题。

## 影响范围

- **仅显示问题**，不影响数据获取、分析逻辑和最终交易决策
- 用户看到错误公司名可能产生困惑，但不影响报告内容的实质准确性
- 仅在 market_report 中出现，其他 6 个分析师报告均使用正确的「英唐智控」

## 修复方案

已在 `tradingagents/agents/analysts/market_analyst.py` 的提示词中增加约束：

```python
- **禁止为股票编造别名/曾用名**：上下文已提供准确的股票名称，只使用该名称，不得自行添加括号别名或曾用名
```

需在下一次实际运行中验证修复效果。

## 验证方法

运行任意 A 股分析，检查 `market_report` 标题是否包含括号别名。若 LLM 仍输出别名，需在 prompt 中加强措辞或增加后处理清理逻辑。
