# v0.3.0 规划

**目标**: 在 v0.2.x 稳定性与数据质量修复的基础上，推进架构优化和功能完善。

---

## 一、政策分析师数据覆盖增强（设计缺失弥补）

### 背景

政策分析师有 3 项数据因接口缺失被标注为 `[数据缺失]`：

| 缺失项 | 要求来源 | 现状 | 可选数据源 |
|--------|---------|------|-----------|
| 宏观政策（降准/降息/LPR/MLF） | 分析师 prompt 要求 | 仅靠 get_news 新闻搜索，命中率低 | 中国人民银行官网、东方财富宏观数据 |
| 监管政策（证监会 IPO/再融资/减持新规） | 分析师 prompt 要求 | 仅靠 get_news 新闻搜索，命中率低 | 证监会官网公告、东方财富监管动态 |
| 地方政策（省/市产业扶持） | 分析师 prompt 要求 | 仅靠 get_news 新闻搜索，命中率低 | 地方政府官网、产业基金公告 |

### 方案选项

| 选项 | 成本 | 收益 | 推荐 |
|------|------|------|------|
| A：保持现状，3 项从「必采」改为「尽力采集」 | 零改动 | LLM 不再标注缺失，减少噪音 | **短期推荐** |
| B：新增 3 个独立数据工具（央行/证监会/地方政策 API） | 高（维护 3 个不稳定源） | 数据完整 | 不推荐 |
| C：新增一个聚合政策新闻工具，定向抓取政策类网站 | 中（1 个新工具） | 覆盖大部分政策需求 | **中期推荐** |

### 建议

- **v0.3.0 短期**：将政策分析师 prompt 中 3 项从「必采」改为「尽力采集」，消除不必要的缺失标记
- **v0.3.0 中期**：调研是否实现一个「政策新闻聚合」工具，定向抓取央行/证监会/工信部/发改委等来源

## 二、市场分析师性能优化（v0.3.0 已完成部分）

### 已完成

- `get_stock_data` 新增 `indicators` 参数，支持 `indicators="all"` 一次性返回 OHLCV + 常用技术指标
- 从 10+ 次工具调用降至 1 次，预计市场分析师耗时从 131s → ~50s
- 提示词已更新，删除 `get_indicators` 工具，LLM 不再独立请求指标
- 别名幻觉 bug 已修复（prompt 增加「禁止编造别名/曾用名」约束）

### 待验证

- 实际运行验证优化效果
- 检查剩余 6 个分析师的耗时是否因工具链简化而同步受益
